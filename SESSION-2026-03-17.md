# Session 2026-03-17: Post-v9 Visual Tuning

## What We Did

### Camera: Wormhole Feel (DONE)
- Removed all camera XY drift (GLSL + JS)
- Removed lookAt wobble

### Brightness Fix (DONE - was the big win)
- **Root cause found**: glow formula used `dist²` (quadratic falloff) — objects at dist=15 were at 5.7% brightness!
- **Fix**: changed to linear falloff `(1.0 + kick*2.0) / (1.0 + dist*0.08)` → objects now at ~45% brightness
- Also boosted: surface brightness multipliers, volumetric glow, near-miss halo, debris/megastructure brightness

### Fog/Color Improvements (DONE)
- Fog color: reduced saturation 0.4→0.3, lightness 0.02→0.015 (less colored wash)
- Fog density: reduced ~30% across all 6 evolution phases
- Fog cap: max 40% opacity (objects always keep 60%+ of their color)
- Buildup fog: reduced closing-in effect
- Vignette: strength 0.45→0.12, minimum brightness floor 0.6 (edges never go below 60%)

### Post-Processing Cleanup (DONE)
- CRT barrel distortion: disabled (was warping edges)
- Chromatic aberration: removed entirely (was splitting objects into RGB copies)
- VHS glitch: disabled (was displacing horizontal strips)
- Feedback trail: heavily reduced (blend 3-8% kicks only, `max(prev, col)` so can only add brightness)
- Feedback edge fade: 3% edge margin fades to zero
- Tone mapping: removed (was compressing bright colors to gray)
- Saturation boost: 1.25→1.35
- Color grading: lightened shadow values, reduced grading strength
- Scanlines: reduced darkening 8%→4%
- Bloom: reduced from 7-tap to 5-tap for performance
- All texture reads now clamped via `sampleScene()`/`samplePrev()` helpers

### Render Pipeline Change (DONE)
- Post pass now renders directly to screen (no blit-to-screen)
- Blit only copies screen→FBO for next frame's feedback
- Added `powerPreference: 'high-performance'` to WebGL context

### New Features (DONE)
- **Volume control**: scroll wheel adjusts volume (0-1)
- **Volume meter**: always visible on right edge, colored bar + energy indicator line
- **Speed boost on kicks**: +4.0 forward speed on kick hits

### Starfield (DONE, from earlier)

## UNRESOLVED: Screen Tearing

### Symptom
- Objects appear "split" along a horizontal line, with top half shifted from bottom half
- Visible on multiple horizontal lines that shift around
- User says it started today (was NOT present before today's session)
- Present from frame 1 (not dependent on audio playing)

### What We Tried (NONE of these fixed it)
1. CRT distortion removal
2. UV clamping on all texture reads (sampleScene/samplePrev helpers)
3. Feedback edge fade
4. Feedback UV margin clamping
5. VHS glitch removal
6. Chromatic aberration removal
7. Rendering post directly to screen (no blit)
8. Bloom reduction (fewer texture reads)
9. `desynchronized: false` on WebGL context (reverted — user has 165Hz monitor)

### What It Might Be
- **Half-res scene rendering**: scene at `w/2 × h/2`, post shader reads it at full-res UVs. If there's an alignment/rounding issue between half-res texels and full-res pixels, objects could appear split. This was always there but maybe a change today exposed it.
- **Performance**: today's changes may have made the shader just heavy enough to miss 165Hz vsync deadline. The glow formula change + sampleScene wrappers + other additions could push frame time over 6ms.
- **Browser rendering bug**: Chrome on Windows with 165Hz display may have compositor issues with WebGL blitFramebuffer.
- **The `sampleScene` clamp at 0.005-0.995**: shifts UVs slightly near edges — probably not the cause since tearing is across the whole screen.
- **Need to investigate**: try running at full resolution (remove the half-res scene) to see if that's the cause. Or try in Firefox/Edge to rule out browser-specific issue.

### Next Steps to Try
1. Test in Firefox/Edge — if no tearing, it's a Chrome/GPU issue
2. Try full-res scene (remove `Math.floor(w/2)`) — if tearing gone, it's the half-res upscaling
3. Check `?debug` FPS counter — if below 165fps, vsync tearing is likely
4. Try adding `gl.finish()` before frame end as a heavy-handed sync
5. Consider rendering scene at 3/4 res instead of 1/2 res
6. Profile: is the scene shader or post shader the bottleneck?
